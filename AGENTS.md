# Agent Guidelines

## Keep Docs Current

After making any change, update the relevant documentation:

- **`Docs/attribution.md` — `What This Fork Adds`**: Add an entry for every new feature or meaningful change made in this fork. If you add a feature, it goes here. If you notice a feature is missing from this list, add it.
- **`Docs/features.md`**: Update or add feature descriptions when behavior changes or new features land.
- **`Docs/api-reference.md`**: Update endpoint docs when backend routes are added, removed, or changed.
- **`Docs/architecture.md`**: Update when the tech stack or project structure changes.
- **`Docs/getting-started.md`**: Update when setup steps, prerequisites, or scripts change.
- **`Docs/keyboard-shortcuts.md`**: Update when shortcuts are added or changed.
- **`README.md`**: Keep the feature bullet list and quick-start in sync with reality. The README stays high-level — detailed content belongs in `Docs/`.

When in doubt: if someone reading the docs after your change would be confused or misinformed, fix the docs before closing the task.
